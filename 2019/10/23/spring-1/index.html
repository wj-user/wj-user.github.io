Spring 5详细示例精通视频

2017.9 Spring 5 -> java 8

优势

方便解耦,简化开发
AOP编程的支持
申明式事务
方便测试
方便继承各种框架
降低javaee API使用难度, 如jdbc

解耦

UI调用service调用dao连接数据库
连接数据库可以使用反射Class.forName()
获取serviceImpl, 可以使用工厂模式获得类新建的实例

可以使用ResourceBundle读取properties文件,获得内容配置等 如账号密码类名. ResourceBundle.getBundle(“config.bean”) //包名.类名,不加后缀properties

ResourceBundle联合工厂模式,可以根据想要的serviceName等获得serviceImpl, dao对象

考虑一下,新建实例并用一个Hashmap存储配置文件中所有类的实例,自己来管理

Spring Bean

bean 存放于core container 核心容器,并根据bean的ID获得对象

ApplicationContext

ApplicationContext是BeanFactory子接口的子接口,特点是创建bean对象时立即加载(读取完xml配置文件后所有bean对象都创建完了,可以在构造函数加上输出查看)
适合单例

BeanFactory

BeanFactory是SpringIOC的顶层接口,特点是创建bean对象时延迟加载(真正要从容器获取bean对象时才会创建,读完配置并不创建)
适合多例

set方法注入

利于复杂类型 

<property> 也是bean的子标签
name    指定的是set方法中的名称,如setName
value    
ref

注入集合数据

结构相同,标签可以互换, 比如List和Map

奇怪的知识

抑制全部警告: @SuppressWarnings(“all”);
p名称空间 xmln:p=”……”
